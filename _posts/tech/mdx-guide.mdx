当ブログで使用している MDX の記法について説明します。

MDX は、Markdown で書いた文章に直接 JSX（React コンポーネント）を埋め込める仕組みです。Markdown（CommonMark）の記法に準拠しています。

また、当ブログでは GitHub Flavored Markdown（GFM）で定義されている機能も利用可能です。

さらに、独自の拡張機能として、リンクカードなどの機能を実装しています。

以下に利用可能な MDX 記法を示します。

## 見出し

```markdown
# 見出し 1

## 見出し 2

### 見出し 3

#### 見出し 4

##### 見出し 5

###### 見出し 6
```

# 見出し 1

## 見出し 2

### 見出し 3

#### 見出し 4

##### 見出し 5

###### 見出し 6

## 装飾

```markdown
**太字（Bold）**
_斜体（Italic）_
**_太字イタリック_**
~~打消し戦（Strikethrough）~~
`インラインコード（Inline Code）`
[テキストリンク](/post/tech/mdx-guide)
本文中の[リンク](/post/tech/mdx-guide)です。
```

## リスト

### リスト

```markdown
- リスト 1
- リスト 2
  - リスト 2-1
  - リスト 2-2
```

- リスト 1
- リスト 2
  - リスト 2-1
  - リスト 2-2

### 番号付きリスト

```markdown
1. リスト 1
2. リスト 2
3. リスト 3
```

1. リスト 1
2. リスト 2
3. リスト 3

## コードブロック

バッククォート 3 つで囲むことで記述できます。シンタックスハイライトには[Shiki](https://shiki.style/)を使用しています。

````markdown
```html
<!DOCTYPE html>
<html lang="ja">
  <head>
    <meta charset="UTF-8" />
    <title>WebSite</title>
  </head>
  <body>
    <p>Test</p>
  </body>
</html>
```
````
